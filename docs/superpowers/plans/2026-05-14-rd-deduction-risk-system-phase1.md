# 研发费用加计扣除风险管理体系 — Phase 1（M0–M6）实施计划

> **For agentic workers / 项目经理:** 本计划为合规 / 组织建设类项目，每步 2–5 分钟内可启动、有明确产出物与责任人。任务以 checkbox (`- [ ]`) 跟踪。Phase 2（M7–M12）与 Phase 3（M13–M24）将在 Phase 1 收官后单独立 plan。

**Goal:** 在 6 个月内（M0–M6）跑通研发费用加计扣除风险管理体系的"治理 + 判定 + 证据 + 内控"四层闭环，输出 20 份核心制度 / 表单 / SOP，飞书多维表格四张主表 MVP 上线，工时填报全员铺开，连续 3 个月月度归集 5 项对账无红灯。

**Architecture:** 治理层（委员会 + 加计办）先立 → 判定层（管理办法 + 判定矩阵 + 三道闸门）落 → 证据层（飞书多维表格四表 + 客观证据强制规则）跑 → 内控层（三级审核 + 月度对账）成。

**Tech Stack:** 飞书多维表格 + 应用引擎（首选）/ Jira / Git / 公司 HR 系统 / 财务 ERP；输出物以 Word / Excel / PDF 制度文件 + 飞书表格为主。

**Spec 引用:** [docs/superpowers/specs/2026-05-14-rd-deduction-risk-system-design.md](../specs/2026-05-14-rd-deduction-risk-system-design.md)
