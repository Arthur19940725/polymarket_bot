## 第一章 总则

**第 1 条 目的**
研发费用加计扣除体系深度依赖个别关键岗位（加计办主任、关键 PM、HR 接口人等）的经验与判断。本 BCP（Business Continuity Plan）方案确保**任一关键岗位离职 / 长期休假 / 突发不可用**时，体系仍可在 7 个工作日内恢复正常运行，避免出现合规真空期。

**第 2 条 BCP 三大原则**
1. **双备份机制**：每个关键岗位都有正式 / 候补两个人选
2. **过程文档化**：核心工作全部 SOP 化，新人 1 周内可上手
3. **早预警 + 早交接**：人员变动征兆出现立即启动应急

**第 3 条 BCP 不适用范围**
- 应对税务稽查的紧急人手不足 → 见《税务稽查应对预案》
- 重大政策变动后的人手不足 → 见《政策变动响应 SOP》
- 公司级业务连续性事件（如自然灾害、疫情）→ 按公司总 BCP

## 第三章 双备份机制

### 第 6 条 双备份的标准

每个关键岗位必须配备：
- **正式岗（A）**：当前实际履职者
- **候补岗（B）**：经过培训、可在 30 天内全面接手的备份人选
- **应急岗（C）**：在 A、B 同时不可用时临时顶岗（短期 7–14 天）

## 第四章 文档化要求

### 第 8 条 核心工作 SOP 化

加计办主任的工作分解为以下 SOP，全部完成可让候补人员在 7 天内基本接手：

| SOP 模块 | 状态 |
|----------|------|
| 月度归集 SOP（含 5 步流程 + 5 项必对） | ✅ 已发布 |
| 季度评审仪表盘 SOP | ✅ 已发布 |
| 立项 / 中检 / 结项闸门审核 SOP | ✅ 已发布 |
| 委员会会议组织 SOP | 待编写（Phase 2）|
| 培训组织 SOP | 待编写（Phase 2）|
| 政策跟踪 SOP | ✅ 已发布 |
| 外部税务师事务所对接 SOP | 待编写（Phase 2）|
| 留存备查档案管理 SOP | 待编写（Phase 2 / M11）|

> **目标：Phase 2 末（M12）SOP 覆盖率 100%。**

### 第 9 条 工作日志要求

关键岗位（特别是加计办主任）必须维护工作日志：
- 月度工作总结（飞书云空间「研发加计扣除管理 / 06-委员会档案 / 月度工作总结」）
- 重大决策的决策依据归档（特别是 B 类项目裁决、政策应对决策）
- 与外部机构（税务局、税务师事务所）的沟通记录
- 关键密码 / 凭证 / 联系方式的安全交接清单（密封保管，仅候补人员 + HR 总监持钥）

### 第 10 条 知识库

加计办在飞书云空间维护「加计办知识库」，包括：
- 体系全景图
- 关键决策案例库
- 常见问题答疑
- 培训资料归档
- 与外部专家的咨询历史

## 第五章 离职 / 不可用预警与处置

### 第 11 条 预警信号

HR + 加计办主任本人 + 加计办主任的直接领导应关注以下信号：
- 加计办主任递交辞职意向 → 立即启动 90 天交接
- 加计办主任长期病假（>14 天）→ 启动 30 天临时方案
- 加计办主任突发离职（紧急） → 启动 7 天应急方案
- 候补人员变动 → 30 天内补足候补
- 关键 PM 离职 → 项目级 BCP

### 第 12 条 90 天交接（计划离职）

| 阶段 | 时长 | 关键任务 |
|------|------|----------|
| 第 1–30 天 | 1 个月 | 候补人员（B）正式确定 + 影子学习 + 工作日志同步 |
| 第 31–60 天 | 1 个月 | B 主导日常工作，A 作为顾问 |
| 第 61–90 天 | 1 个月 | B 独立运行，A 离职前最后阶段答疑 |
| 离职日 | 当天 | 全部交接清单完成 + 委员会备案 + 密码 / 凭证回收 |

### 第 13 条 30 天临时方案（长期休假 / 病假）

- 候补人员（B）接手日常工作
- 重大决策（如委员会议事、对外答复）延迟或由应急岗（C，财务总监）兼任
- 加计办主任远程参与必要决策（如可）

### 第 14 条 7 天应急方案（突发离职）

```
T+0：CFO / HR 接到通知
       ↓
T+1：候补人员（B）正式接管
       - 接收工作邮箱
       - 接收飞书账号
       - 接收档案库钥匙
       - 接收外部联系方式
T+2：紧急召开委员会会议，介绍 B 给委员
       ↓
T+3–T+7：B 在加计办其他成员协助下熟悉环境
       ↓
T+7 末：B 主持首次月度归集对账会（标志正式上岗）
```

## 第六章 交接清单（关键岗位通用）

### 第 15 条 交接物清单

加计办主任离职 / 长期休假前，必须按以下清单完成交接：

#### A. 文档与制度
- [ ] 全部制度文件电子版交接
- [ ] 边界案例库（飞书表 + Excel 备份）
- [ ] 历年自查报告 + 委员会评审纪要
- [ ] 工作日志（电子 + 纸质）
- [ ] 培训材料库

#### B. 数据与系统
- [ ] 飞书多维表格 admin 权限
- [ ] 飞书云空间访问权限
- [ ] Jira / GitLab 接入凭证（与 IT 协调更换）
- [ ] HR 系统对接通道
- [ ] 财务 ERP 查阅权限

#### C. 外部联系
- [ ] 税务局对接联系人
- [ ] 外部税务师事务所对接联系人 + 现有合同
- [ ] 律师事务所对接联系人（如有）
- [ ] 科技局对接联系人

#### D. 内部联系
- [ ] 各部门接口人通讯录
- [ ] 全体 PM 通讯录
- [ ] 委员会成员通讯录
- [ ] 关键供应商 / 合作方通讯录

#### E. 决策与判例
- [ ] 全部 B 类项目委员会裁决纪要
- [ ] 在办应急事项（如有）的完整背景
- [ ] 已识别但未处置的风险事项清单
- [ ] 未完成的政策影响评估事项

#### F. 凭证与密码
- [ ] 加计办档案库钥匙（纸质）
- [ ] 各系统密码（密封移交给 HR 总监）
- [ ] 印章（如适用）

### 第 16 条 交接验收

交接完成后，CFO + HR 总监 + 接任人（B）三方签字确认：
- ☐ 全部清单已交接
- ☐ 接任人已熟悉
- ☐ 委员会备案

## 第七章 培训与演练

### 第 17 条 候补人员培训

候补人员（B）应在指定后 60 天内完成以下培训：
- ☐ 体系全景培训（≥4 课时）
- ☐ 月度归集实操（≥2 次跟岗）
- ☐ 委员会会议旁听（≥2 次）
- ☐ 与外部税务师事务所对接（≥1 次）
- ☐ 应急预案演练（≥1 次）

### 第 18 条 年度 BCP 演练

每年 12 月（汇算清缴后 + 春节前），加计办组织 1 次 BCP 演练：
- 模拟加计办主任突发离职
- 候补人员（B）独立完成一次月度归集
- 评估候补人员独立运行能力
- 出具《BCP 演练评估报告》

---

## 第八章 候补人员激励

### 第 19 条 候补津贴

候补人员承担额外责任，公司给予适当激励：
- 候补人员（B）：年度津贴 6000–12000 元
- 应急岗（C，临时顶岗超 7 天）：临时津贴日均 200–500 元
- 实际接手且独立运行的：按正式岗位薪酬调整

### 第 20 条 候补人员考核

候补人员的年度考核中纳入：
- 培训完成度
- 演练表现
- 实际接手能力（如有）

---

## 第九章 数据与文件保护

### 第 21 条 关键人员离职后

- 工作邮箱保留 6 个月（重要邮件归档）
- 飞书账号转交，历史聊天记录归档
- 关键档案双备份（电子 + 纸质）
- 关键密码立即修改

### 第 22 条 防止"知识带走"

- 关键岗位人员日常工作必须文档化（飞书 + 加计办知识库）
- 重要决策必须有书面归档（不能仅凭个人记忆）
- 离职前 60 天不再分配新的"个人决策"任务，改为"协助 + 培训"模式

## 第十章 BCP 治理

### 第 23 条 责任分工

| 任务 | 责任人 |
|------|--------|
| BCP 方案维护 | 加计办主任 |
| 候补人员任命 | 加计办主任 + 所在部门负责人 + CFO 备案 |
| 候补培训 | 加计办主任 |
| 年度演练 | 加计办主任 + HR |
| 紧急交接组织 | CFO + HR |
| BCP 经费 | 加计办预算 |

### 第 24 条 BCP 经费

| 项目 | 年度预算（万元）|
|------|------------------|
| 候补人员津贴 | 5–10 |
| 培训与演练 | 3–8 |
| 文档与知识库维护 | 2–5 |
| **合计** | **10–23** |

### 第 25 条 BCP 季度审视

加计办主任每季度向委员会汇报 BCP 状态：
- 关键岗位备份状态
- 候补培训进展
- 离职 / 病假等风险信号
- 文档化覆盖率

## 第十一章 附则

**第 26 条** 本方案与公司总 BCP 政策冲突时，以公司总 BCP 政策为准；与《管理办法》冲突时，以本方案为准。

**第 27 条** 本方案的解释权归委员会，日常解释由加计办主任行使。

**第 28 条** 本方案自 2026-12-15 起生效。
