## ⚠️ 紧急启动指引

**如果你正在面对税务稽查或约谈**，请按以下步骤立即行动：

1. **保持冷静，不要慌乱回答任何问题**。礼貌请求对方留下联系方式与具体诉求。
2. **1 小时内拨打加计办主任手机**：【___】
3. **在加计办指导下**才接听后续电话或回复邮件。
4. **不要单独签署任何文件**——所有书面回复必经委员会 + 法务复核。
5. **不要现场补签、修改、销毁任何文件**——这会从合规问题升级为刑事问题。

> **黄金 24 小时**：从收到通知到出具《应对策略备忘》，须在 24 小时内闭环。

## 第一章 总则

**第 1 条 目的**
为应对税务局对研发费用加计扣除的稽查、约谈、问询、调查等事项，建立标准化、可执行、有纪律的应对预案，保护公司合法权益、避免不当处置导致损失扩大。

**第 2 条 三大原则**
1. **冷静 + 纪律**：任何环节不得擅自答复、补签、修改文件
2. **专业先行**：所有书面回复经加计办 + 法务 + 必要时外部税务师三方把关
3. **不卑不亢**：依法配合 + 据理力争，加计扣除合规性的举证责任由公司承担，但稽查指控的事实由税务局承担

**第 3 条 适用范围**
本预案适用于：
- 税务局针对研发费用加计扣除的**专项稽查**
- 税务总局 / 省局 / 市局的**风险约谈**
- 税务事项通知书 / 询问函 / 限期改正通知书
- 上门检查、现场询问、调阅档案
- 涉及股权变更 / IPO / 并购重组中的研发费用合规审查

**第 4 条 不适用范围**
- 普通的发票认证 / 代开 / 申报咨询（按日常业务流程）
- 与研发费用无关的稽查（按公司其他预案）
- 涉嫌刑事犯罪的调查（直接转交司法机关，本预案不适用）

## 第二章 四类应急场景

### S1：上门稽查 / 现场询问 🔴🔴🔴

**特征**：税务稽查局执法人员（≥2 人）持《税务检查证》上门，要求查阅资料、询问员工、复印文件。

**核心动作**：
- 立即通知加计办主任 + 法务 + 财务总监到场
- 礼貌接待，提供独立会议室
- 询问检查依据（《税务检查证》《税务事项通知书》编号）
- 不主动提供未被要求的资料
- 全程有公司代表在场陪同

### S2：电话约谈 🔴🔴

**特征**：税务局打电话、发短信、邮件，要求公司财务总监或法人代表到税务局接受约谈。

**核心动作**：
- 接听人记录：来电号码 + 对方姓名 + 部门 + 约谈事项 + 约谈时间地点
- 礼貌请求"我需要与公司财税团队沟通后给您确认时间"，争取 24–48 小时缓冲
- 立即通知加计办主任 + CFO
- 内部准备工作（见 §四 黄金 24 小时清单）

### S3：税务事项通知书 / 询问函 🔴

**特征**：邮政 / EMS 收到税务局正式书面通知，要求限期（一般 7–15 个工作日）提交资料 / 出具说明 / 完成自查。

**核心动作**：
- 收件人 1 小时内扫描通知书发给加计办主任
- 加计办主任 24 小时内研判，提出初步应对方案
- 委员会审议后正式回复

### S4：日常风险评估 / 数据比对 🟡

**特征**：税务局通过金税系统大数据发现公司加计扣除指标异常（如占比畸高、跨年度波动大），发出非正式的风险提示。

**核心动作**：
- 不立即触发紧急预案，但加计办需 5 个工作日内自查
- 出具《自查应对方案》报委员会备案

## 第三章 应急组织架构

### 第 5 条 应急小组（任何 S1–S3 触发后立即成立）

| 角色 | 担任人 | 主要职责 |
|------|--------|----------|
| 组长 | **CFO** | 决策；对外正式签字；与税务局高层沟通 |
| 副组长 | **加计办主任** | 内部协调；技术性问题答复；资料调取 |
| 法务代表 | **法务总监** | 法律意见；书面回复审核；权利保护 |
| 财务代表 | **财务总监** | 财务数据；申报口径解释 |
| 研发代表 | **研发 VP 或指定 PM** | 研发活动技术细节；技术不确定性说明 |
| 外部专家 | **外部税务师事务所合伙人** | 政策咨询；涉及重大金额时正式聘请 |
| 内审 | **内审负责人** | 独立监督；档案见证 |

### 第 6 条 决策机制

- 一般事项：副组长（加计办主任）+ 法务 + 财务三方一致即可
- 重大事项（涉及金额 >500 万 / 罚款风险 / 主动更正申报）：必须组长（CFO）签字
- 重大对外答复（书面回复 / 现场陈述）：必须组长 + 法务 + 副组长三方会签

## 第四章 ⭐ 黄金 24 小时操作清单

> **使用方法**：从收到通知 / 约谈 / 上门的 T+0 时点开始计时，按以下清单逐项执行。

### T+0 ~ T+1 小时：信号识别与启动

- [ ] **T+0+15 分钟**：收信人（前台 / 助理 / 财务 / CFO）识别为税务稽查事项
- [ ] **T+0+30 分钟**：收信人电话 / 飞书 / 短信通知 **加计办主任**（手机号见附录 A）
- [ ] **T+0+45 分钟**：加计办主任电话通知 **CFO**
- [ ] **T+0+60 分钟**：CFO 决定是否启动应急小组（一般 S1/S2/S3 应启动）

### T+1 ~ T+2 小时：应急小组成立

- [ ] 加计办主任建立飞书私密群「加计扣除应急-YYYYMMDD」
- [ ] 拉入应急小组全部成员（含外部税务师，如需）
- [ ] 群内首次会议：明确事项、初步分工、保密纪律
- [ ] 加计办主任启动「应急事项工单」编号 `EM-YYYYMMDD-NNN`

### T+2 ~ T+4 小时：初步评估

- [ ] 法务 + 加计办主任研判 **稽查范围**：
  - ☐ 涉及年度（最近 1 年 / 3 年 / 5 年）
  - ☐ 涉及项目（具体项目编码 / 整体）
  - ☐ 涉及金额（估算）
  - ☐ 涉及部门（仅财务 / 财务 + 研发）
- [ ] 出具《初步评估备忘》（≥1 页，加计办主任签字）

### T+4 ~ T+8 小时：调取留存备查档案

- [ ] 加计办按 **10 类档案清单** 逐项检查齐备性
- [ ] 重点档案优先调取：
  - ☐ 立项书 + 技术不确定性说明（涉及项目）
  - ☐ 委外 / 合作研发合同 + 科技局登记证明
  - ☐ "研发支出"辅助账与汇总表
  - ☐ 工时台账 + 客观证据导出包
  - ☐ 历年自查报告 + 委员会评审纪要
- [ ] 缺失档案立即标记（**严禁现场补做**）
- [ ] 飞书云空间 / 加计办档案库双向核对

### T+8 ~ T+16 小时：法务与外部专家会商

- [ ] 法务总监审视：
  - ☐ 稽查依据的法律合规性
  - ☐ 公司是否享有抗辩权利
  - ☐ 是否涉及主动更正申报的法律风险
- [ ] 涉及金额 >500 万 / 罚款风险 → 正式聘请 **外部税务师事务所**（24 小时内签 NDA + 委托合同）
- [ ] 外部专家初步意见

### T+16 ~ T+20 小时：拟定应对策略

- [ ] 应急小组会商，决定：
  - ☐ 答复路径（口头 / 书面 / 现场提交）
  - ☐ 关键问题回答口径
  - ☐ 是否主动更正申报
  - ☐ 是否需聘请律师
  - ☐ 是否需要向税务局申请延期 / 听证

### T+20 ~ T+24 小时：出具《应对策略备忘》

- [ ] 备忘文档结构（5 页内）：
  - 一、事项摘要（200 字）
  - 二、风险评估（含金额 / 法律 / 声誉）
  - 三、应对方案（3 套备选 + 推荐方案）
  - 四、时间节点与责任人
  - 五、附件清单
- [ ] **应急小组组长（CFO）签字** + 委员会全员知情

### T+24 小时：上报董事长 + 启动正式应对

- [ ] CFO 上报董事长（如涉及金额 >1000 万 / 罚款风险 / 声誉风险）
- [ ] 进入正式应对阶段（详见 §五）

## 第五章 ⭐⭐⭐ 三道防线（抗辩核心）

### 第一道防线：留存备查档案完备性（70% 胜算）

**核心**：稽查现场出示完整的 10 类档案，证明公司体系化运行。

**关键档案**：
1. 立项书 + 技术不确定性说明（A 类项目 ≥500 字 / B 类 ≥800 字）
2. 研发人员名单 + 月度工时台账 + 客观证据导出包
3. 委外 / 合作研发合同 + 科技局登记证明
4. 集中研发项目分摊明细（如适用）
5. "研发支出"辅助账与汇总表
6. 项目结项报告 + 验收证明
7. 历年自查报告 + 整改记录
8. 委员会评审纪要
9. 外聘人员 + 仪器设备 + 无形资产费用分配说明
10. 省局规定的其他资料

**应答话术**：
> "我们公司从 2026 年起建立了完整的研发费用加计扣除风险管理体系，覆盖 10 类留存备查档案，按公司《管理办法》执行。下面我们逐类向您展示。"

### 第二道防线：技术不确定性说明（核心抗辩证据）

**核心**：A/B 类项目立项书中的"技术不确定性说明"是判定为研发活动的核心证据。

**应答话术示例**：
> "稽查员同志，关于这个推荐系统项目为什么算研发，我可以请研发负责人来具体说明。
>
> 关键证据是项目立项时的技术不确定性说明：当时多模态推荐方案在电商场景没有成熟先例，业界 SOTA 模型在通用领域有效但电商场景未知。我们做了 4 周 POC，对比了三套技术方案，才确定主路线。如果稽查员需要查看技术方案设计文档、POC 实验记录、Git 仓库历史，我们都可以提供。"

**关键纪律**：研发代表说话要客观，不夸大、不贬低。

### 第三道防线：边界案例库 + 委员会裁决记录（体系性证明）

**核心**：证明公司有系统化的判定机制，不是临时拼凑。

**关键档案**：
- 边界案例库（≥30 例时强力）
- B 类项目的委员会专题裁决纪要
- 季度风险评审仪表盘记录

**应答话术**：
> "我们对于研发 vs 非研发的边界有系统化的判定。所有 B 类（准研发）项目都经过委员会的专题裁决，记录在边界案例库里。本次稽查涉及的 XX 项目，是 2026-XX 月委员会审议通过的，纪要附件可调取。"

## 第六章 严禁事项（一票否决红线）

### 第 7 条 **绝对禁止**

- ❌ 现场临时编造、伪造、补签、倒签任何文件
- ❌ 销毁、隐匿、修改任何留存备查档案
- ❌ 串供、向其他员工传授"回答口径"
- ❌ 未经加计办 + 法务同意单独答复
- ❌ 向稽查员行贿、请客、送礼
- ❌ 单独签署任何书面材料（必经委员会签批）
- ❌ 拒绝提供应当提供的资料
- ❌ 对稽查员撒谎或恶意误导

> 上述任一行为，可能将税务合规问题升级为刑事问题（伪证罪 / 行贿罪 / 妨碍公务罪等），后果极其严重。

### 第 8 条 必须做的

- ✅ 依法配合 + 礼貌接待
- ✅ 所有沟通有记录、有见证
- ✅ 不确定的问题"我需要核实后给您正式回复"
- ✅ 涉及数字 / 金额 / 政策口径的，引用文件原文
- ✅ 留存稽查全过程的录像（应急小组在场情况下）/ 录音（如允许）
- ✅ 每天结束当日工作时由应急小组复盘

## 第七章 答复路径与文书规范

### 第 9 条 口头答复

- 仅限"程序性问题"（如时间地点确认）+ 已演练的标准答复
- 涉及实质性问题一律：**"这个问题我需要回去核实后给您正式书面回复"**
- 记录原则：当事人 + 见证人 + 记录人三方在场

### 第 10 条 书面答复

- **所有书面答复**：先加计办起草 → 法务复核 → 委员会签批 → CFO 签发
- 模板见附录 B
- 字数原则：能短则短；多说多错
- 引用文件 / 政策时必精确（年号 / 文号 / 条款）

### 第 11 条 现场提交资料

- 准备 2 套：1 套交税务、1 套留存
- 每份标注页码 + 章戳 + 加计办主任签名
- 提交清单一式两份，税务签收

### 第 12 条 申请延期

如时间紧迫，可向税务局申请延期（一般可获得 7–15 个工作日）：
- 标准话术："我们需要更充分的时间整理资料以准确答复，请允许我们 X 个工作日内提交。"
- 申请延期不构成不配合，只要理由合理、申请及时

## 第八章 与监管沟通的纪律

### 第 13 条 接待礼仪

- 主接待人：加计办主任 + 法务（必须两人在场）
- 称呼："X 检查员" / "X 同志"
- 接待室：独立、隔音、有监控
- 茶水：常规礼貌即可，不过分热情
- 资料：纸质 / 电子双备份

### 第 14 条 应对策略 4 字诀

- **冷**：保持冷静，不急于答复
- **稳**：节奏稳，不慌不忙
- **准**：依据准确，引用政策原文
- **诚**：依法配合，态度诚恳

### 第 15 条 严禁言论

❌ 不说："这个我们也搞不清楚 / 反正同行都这么做 / 上次审计也没说什么"
❌ 不说："您看这个能不能通融一下 / 我们关系不错"
❌ 不说："我们董事长在 XX 单位有关系"
❌ 不说："如果有问题我们就改 / 罚多少都行"

✅ 应说："这是我们的留存备查档案，请您查看。"
✅ 应说："这个问题我们需要核实后正式书面回复。"
✅ 应说："我们公司一向严格执行国家税收政策，如果您发现任何问题，我们一定彻底整改。"

## 第九章 主动更正申报

### 第 16 条 触发情形

发现以下情形之一，应考虑主动更正申报：
1. 自查发现历史申报存在明显瑕疵（金额可量化）
2. 稽查问询前主动发现的错误
3. 政策变动导致历史申报口径需调整

### 第 17 条 决策路径

```
发现瑕疵
   ↓
加计办评估金额 + 性质
   ↓
   ├── 金额 <100 万 + 性质明确 → 财务总监决策 → 直接更正申报
   ├── 100–500 万 → 委员会决策
   └── >500 万 → 委员会 + 外部税务师 + 必要时律师
   ↓
执行更正
   ↓
留存全过程记录
```

### 第 18 条 主动更正的优势

- 法律上：避免被税务局发现后定性为偷税（罚款 0.5–5 倍）
- 实务上：滞纳金按日万分之五计算，比罚款轻得多
- 声誉上：体现公司自我合规能力

### 附录 D：常见问题速查（FAQ）

**Q1：稽查员要求复印我们的工资表，可以给吗？**
A：可以，但只提供已脱敏的研发人员部分（与稽查事项相关）。非相关员工的工资数据是公司商业秘密，可拒绝。

**Q2：稽查员要求约谈某位研发员工，可以拒绝吗？**
A：原则上不能拒绝，但有权安排在公司会议室、由法务陪同。约谈前 5 分钟应给员工简短指导："如实回答你知道的，不知道就说不知道，不要猜测。"

**Q3：稽查员说"看你们配合不积极，要罚款"，怎么办？**
A：礼貌回应："我们一直在依法配合，并按规定时限提交资料。如果贵局认为有应当提供但未提供的资料，请书面列明，我们会及时补充。"

**Q4：发现某历史年度有瑕疵，该立即承认吗？**
A：**不要现场承认。** 立即上报应急小组，按本预案第九章流程评估后再决定。

**Q5：稽查员暗示需要"协调"（行贿暗示），怎么办？**
A：礼貌但坚决拒绝。事后立即向 CFO + 内审 + 法务报告。必要时录音作为证据保护公司。
