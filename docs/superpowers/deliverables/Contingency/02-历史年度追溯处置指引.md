## 第一章 总则

**第 1 条 目的**
当公司内部自查、内审、外审或税务稽查发现历史年度的研发费用加计扣除存在瑕疵时，本指引为加计办、委员会决策提供标准化处置路径，控制风险敞口、避免损失扩大。

**第 2 条 适用范围**
适用于公司已申报但发现存在以下情形之一的历史年度加计扣除事项：
1. 工时记录不实或证据链缺失
2. 项目认定有误（应属 C / D 类的被归为 A / B 类）
3. 留存备查档案缺失或不齐
4. 委外 / 合作研发合同瑕疵导致计入金额可能不合规
5. "其他相关费用"5% 限额超限被发现
6. 与高新 / 软件企业认定口径打架被外部审计师质疑
7. 因政策变动导致历史申报口径需调整
8. 自查或抽样审计发现的其他重大瑕疵

**第 3 条 原则**
1. **早发现、早处置**：发现即上报，5 个工作日内出具《风险事项报告》
2. **科学评估**：金额、性质、可挽回性三维度综合评估
3. **依法处置**：在法律允许范围内最小化损失
4. **闭环管理**：处置完成后纳入体系改进，避免重蹈覆辙

## 第三章 处置决策三条路径

### 路径 ① · 主动更正申报（推荐场景：金额小、瑕疵明确）

**适用情形**：
- 单一事项金额 <100 万元
- 瑕疵性质明确（如工时记录某条不实、某项目应剔除）
- 距汇算清缴申报年度 ≤ 3 年（追溯时效内）
- 公司自主发现，尚未被税务局问询

**处置流程**：
- [ ] 加计办主任 5 个工作日内出具《主动更正申报方案》
- [ ] 财务总监决策签发
- [ ] 在下一年度汇算清缴时调整申报
- [ ] 留存全过程记录归档

**优势**：
- 避免被税务局发现后定性为偷税（罚款 0.5–5 倍）
- 滞纳金按日万分之五计算（成本可控）
- 体现公司自我合规能力，对税务局形成正面印象
- 通常不影响公司纳税信用评级

**劣势**：
- 需缴纳滞纳金
- 需自行承担纠正成本（审计 / 申报）

### 路径 ② · 暂不处置，等待税务局检查（推荐场景：金额不大、瑕疵模糊）

**适用情形**：
- 单一事项金额 100–500 万元，**且性质判断模糊**（可能是合规的、也可能不合规）
- 瑕疵涉及主观判断争议（如某项目算 A 类还是 B 类）
- 历史档案虽不完整但有间接证据可补
- 公司已能完善内部记录，避免税务局发现

**处置流程**：
- [ ] 加计办出具《风险事项评估报告》
- [ ] **委员会专题会议决策**（必经流程）
- [ ] 内部不主动申报，但：
  - 完善内部记录（补充证据 / 整理档案）
  - 准备"如被问询"的抗辩材料
- [ ] 关注税务局动态（金税系统大数据风险评估）
- [ ] 留存决策依据，证明非"主观故意隐瞒"

**优势**：
- 不主动产生滞纳金
- 不暴露公司其他可能的瑕疵
- 给公司体系建设留出时间窗口

**劣势**：
- 仍有被发现的风险，被发现后处罚可能加重
- 增加心理与组织负担
- 委员会决策需慎重，决策文档需归档以备未来举证

### 路径 ③ · 聘请外部专家深度评估（必选场景：金额大、定性争议）

**适用情形**：
- 单一事项或累计金额 >500 万元
- 涉及税务部门可能定性偷税 / 骗税的争议
- 历史档案严重缺失
- 涉及股权变更 / IPO / 并购重组中的合规审查
- 已有税务局问询，但金额超出公司内部应对能力

**处置流程**：
- [ ] 应急小组成立（与稽查应对预案共用同一架构）
- [ ] 法务总监 24 小时内启动外部税务师事务所 + 必要时律师事务所采购流程
- [ ] 与外部专家签订 NDA + 委托合同（建议金额 30–100 万）
- [ ] 外部专家出具《专业意见书》
- [ ] 委员会基于专业意见决策
- [ ] CFO 签发最终处置方案，必要时报董事会

**优势**：
- 专业意见可作为后续抗辩证据
- 律师 / 税务师事务所执业保险可分担风险
- 决策的合规性与正当性获得外部背书

**劣势**：
- 外部专家费用高（30–100 万）
- 处置周期长（一般 1–3 个月）
- 可能暴露公司其他历史问题

┌─────────────────┬────────────────┬─────────────┬───────────────┐
│ 瑕疵明确         │  路径①         │  路径①+②   │  路径②+③    │
│ 自查发现         │  主动更正      │   委员会决策 │  外部专家+委员会│
├─────────────────┼────────────────┼─────────────┼───────────────┤
│ 瑕疵模糊         │  路径②         │   路径②     │  路径③       │
│ 自查发现         │  暂不处置+完善 │  委员会决策 │  外部专家+委员会│

## 第五章 关键判断要点

### 第 4 条 金额判断

- 单一事项金额：该项目 / 该年度涉及的加计扣除金额
- 累计金额：跨年度同性质瑕疵累计
- 节税影响：金额 × 25%（公司适用所得税率）
- 罚款风险：节税影响 × 0.5–5 倍（按偷税情节）+ 滞纳金（按日万分之五）

### 第 5 条 性质判断

- **明确瑕疵**：可量化、有客观证据（如工时无 Jira/Git 关联）
- **模糊瑕疵**：涉及主观判断、专业争议、政策模糊地带
- **故意偷税**：构成《税收征收管理法》第 63 条的情形（永远不要承认）

### 第 6 条 时效判断

- 增值税：3 年（一般情形）/ 5 年（重大情形）
- 企业所得税：3 年（一般）/ 5 年（重大）/ 无限期（偷税）
- 因税务机关原因延误：可顺延

### 第 7 条 影响范围判断

- 是否影响其他年度（递延瑕疵）
- 是否影响其他税种（如增值税 / 个税）
- 是否影响其他申报（高新认定 / 软件企业认定）
- 是否影响公司声誉 / IPO / 融资

## 第七章 决策流程

```
发现风险事项
       ↓
加计办主任 5 日内出具《风险事项报告》
       ↓
   按金额分级：
       │
       ├── < 100 万 + 瑕疵明确 → 财务总监决策 + 路径①
       │
       ├── 100–500 万 / 瑕疵模糊 → 委员会专题会议决策
       │
       └── > 500 万 / 重大争议 → 委员会 + 外部税务师 + 必要时律师
                                  ↓
                              CFO 签发最终方案
                                  ↓
                             （重大项报董事会）
       ↓
执行处置
       ↓
留存全过程记录（10 年）
       ↓
体系改进（更新判定矩阵 / 增加培训点 / 修订制度）
       ↓
事项关闭
```

## 第八章 与稽查应对预案的衔接

| 时点 | 决策 |
|------|------|
| 自查发现，尚未被稽查 | 适用本指引，决定是否主动更正 |
| 自查发现，已知税务局可能稽查 | 同时启动本指引 + 稽查应对预案 |
| 已收到税务局问询 | 优先启动稽查应对预案，本指引作为评估工具 |
| 稽查后被定性 | 已超出本指引范围，进入行政复议 / 诉讼流程 |

## 第九章 体系改进闭环

每个历史年度追溯事项处置完成后，加计办主任组织以下闭环工作：

- [ ] 在**边界案例库**中录入本事项（作为后续判定参照）
- [ ] 更新**判定矩阵**（如有新边界共识）
- [ ] 增加针对性培训（PM / 财务 / 全员，根据原因）
- [ ] 修订相关制度（如发现是制度漏洞）
- [ ] 在季度评审仪表盘中跟踪同类问题趋势

## 第十章 重要纪律

### 第 8 条 严禁

- ❌ 私下隐瞒已发现的瑕疵不上报
- ❌ 未经委员会决策擅自主动更正申报
- ❌ 销毁、隐匿、伪造任何相关档案
- ❌ 与发现方（外审 / 内审）串通遮掩
- ❌ 未经法务审核擅自与税务局沟通

### 第 9 条 必须

- ✅ 5 日内出具风险事项报告
- ✅ 按金额分级提交决策
- ✅ 全程留痕
- ✅ 决策依据归档（包括"暂不处置"的决策也要归档）
- ✅ 处置完成后做体系改进

## 第十一章 附则

**第 10 条** 本指引与《税务稽查应对预案》《研发费用加计扣除管理办法》冲突时，以更严格规定为准。

**第 11 条** 本指引的解释权归委员会，日常解释由加计办主任行使。

**第 12 条** 本指引自 2026-12-15 起生效。
