## 第一章 总则

**第 1 条 目的**
研发费用加计扣除政策由财政部、税务总局、科技部等部门多头管理，每年都有新文件、新口径出台。本 SOP 用于规范公司对政策变动的监测、评估、宣贯、体系适配，确保**14 天内完成内部影响评估**。

**第 2 条 监测对象**

| 监测对象 | 来源 | 重要级 |
|----------|------|--------|
| 财政部、税务总局发布的研发加计扣除政策文件 | 总局官网 / 12366 | 🔴 |
| 总局公告（口径调整、操作指南）| 总局公告系统 | 🔴 |
| 省、市税务局的地方实施细则 | 主管税务机关 | 🟡 |
| 科技部 / 工信部 关于研发活动认定的文件 | 部委官网 | 🟡 |
| 高新认定 / 软件企业认定 政策更新 | 科技部 / 工信部 | 🟡 |
| 财政部、税务总局的解释 / 答疑 | 12366 / 答疑会 | 🟢 |
| 行业协会、税务师事务所的政策解读 | 业内媒体 | 🟢 |

---

## 第二章 四步响应机制

```
   政策发布 ──→ 14 天 ──→ 30 天 ──→ 60 天
        │         │           │           │
        ▼         ▼           ▼           ▼
   实时监测   影响评估    内部宣贯   体系修订
   （加计办） （加计办+顾问）（加计办）  （委员会）
```

---

## 第三章 步骤 1：政策实时监测

### 第 3 条 监测渠道

加计办主任为政策监测的责任人，需订阅以下渠道：

| 渠道 | 频率 | 操作 |
|------|------|------|
| 国家税务总局官网"政策法规" | 每日 | RSS / 邮件订阅 |
| 12366 纳税服务平台 | 每日 | 微信公众号关注 |
| 财政部官网"政策发布" | 每日 | 邮件订阅 |
| 科技部 / 工信部官网 | 每周 | 浏览 |
| 主管税务机关网站 + 微信 | 每日 | 关注 |
| 外部税务师事务所政策快讯 | 实时 | 邮件订阅 |
| 行业税务师协会通讯 | 每月 | 邮件订阅 |

### 第 4 条 监测记录

加计办主任每周一在飞书「政策跟踪日志」更新一次：
- 本周关键政策变动
- 已识别的影响事项
- 待评估事项清单

每月在季度评审仪表盘附"政策动态摘要"。

## 第四章 步骤 2：影响评估（14 天）

### 第 5 条 启动条件

发现以下政策变动后**24 小时内**启动正式影响评估：
1. 中央层面发布的研发加计扣除相关公告 / 通知
2. 主管税务机关发布的本地实施细则
3. 总局答疑或案例中出现的口径调整
4. 高新认定 / 软件企业认定 中涉及研发费用定义的变动

### 第 6 条 评估流程

```
T+0：政策发布
T+1：加计办主任研判，决定是否启动正式评估
T+3：加计办起草《政策变动影响评估初稿》
T+5：与外部税务师事务所（合作单位）会商
T+7：评估初稿提交财务总监 + 法务总监复核
T+10：必要时召开委员会专题会议
T+14：评估终稿提交委员会，CFO 签发
```

## 第五章 步骤 3：内部宣贯（30 天）

### 第 8 条 宣贯对象与内容

| 对象 | 内容 | 形式 |
|------|------|------|
| 委员会 | 评估报告全文 + 应对建议 | 专题会议 |
| 加计办 + 财务 | SOP 执行细节 | 内部培训 |
| 各 PM | 项目层影响 + 操作变化 | 飞书群 + 培训 |
| 研发员工 | 工时填报 / 立项流程的变化（如有）| 飞书公告 |
| 全员（重大变动）| 政策变动摘要 + 影响 | 邮件 + 工作台横幅 |

### 第 9 条 宣贯节奏

- T+15：委员会专题会议通过应对方案
- T+18：加计办出具《政策变动宣贯材料》
- T+20–25：分场次培训
- T+30：宣贯出勤率核查（目标 ≥95%）

### 第 10 条 宣贯材料

每次政策变动宣贯应包括：
- 政策变动对照表（变更前 vs 变更后）
- 操作变化点（对员工 / PM / 财务 / 加计办的具体影响）
- Q&A 5–10 题
- 培训 PPT

## 第六章 步骤 4：体系修订（60 天）

### 第 11 条 修订对象

依据评估报告，识别需要修订的制度文件：

| 文件类型 | 常见修订点 |
|----------|-----------|
| 管理办法 | 政策依据条款 / 加计比例 / 费用归集范围 |
| 判定矩阵 | A/B/C/D 分类边界 |
| 立项 / 中检 / 结项模板 | 字段调整 |
| 工时填报指南 | 操作要求 |
| 月度归集 SOP | 归集口径 |
| 季度评审仪表盘 | 指标定义 / 阈值 |
| 留存备查档案管理办法 | 档案种类 |

### 第 12 条 修订流程

```
T+30：加计办主任列出待修订文件清单
T+35：加计办分头起草修订稿
T+45：法务 + 财务 + 外部税务师复核
T+50：委员会审议
T+55：CFO 签发新版本
T+60：飞书云空间更新，旧版本归档；同步全员通知
```

### 第 13 条 版本管理

- 制度文件版本号：v1.x → v2.0（重大政策导致）/ v1.1 → v1.2（小幅调整）
- 每次修订均归档前一版本，保管期 10 年
- 在制度文件首页"版本历史"表中记录修订原因 + 日期 + 主要变化

## 第七章 申报口径调整

### 第 14 条 跨年度影响处置

政策变动可能影响已申报年度或当年度申报：

| 影响 | 处置 |
|------|------|
| 仅影响新发生业务 | 自政策生效日起按新口径 |
| 影响当年度尚未申报业务 | 立即按新口径，必要时调整年度预算 |
| 影响已申报但未汇算清缴 | 评估是否需要在汇算清缴时调整 |
| 影响已汇算清缴年度 | 评估是否需要走主动更正申报（依《历史年度追溯处置指引》）|

### 第 15 条 与汇算清缴的衔接

- 每年 4–5 月汇算清缴前 30 天，加计办主任全面盘点过去 12 个月的政策变动
- 出具《本年度汇算清缴口径备忘》，明确各项政策的适用版本
- 委员会审议通过后作为申报依据

## 第八章 与主管税务机关沟通

### 第 16 条 主动沟通机会

加计办应建立与主管税务机关的常态化沟通渠道：
- 每年 1 次的纳税企业座谈会
- 每年 2 次的政策辅导班
- 重大政策发布后的答疑场次
- 必要时的专项咨询（书面）

### 第 17 条 沟通纪律

- 沟通由加计办主任 + 财务总监主导
- 涉及具体项目认定的咨询事项，需事先经委员会审议
- 与税务机关的所有书面往来归档保管

---

## 第九章 信息安全

### 第 18 条 内部敏感性

政策评估报告涉及公司财务敏感信息（节税金额、风险敞口等），密级为内部 / 机密：
- 评估报告：仅委员会 + 加计办持有完整版
- 摘要版：可对加计办内部 + 财务部 + 法务部分发
- 公开版：可对全员宣贯，不含敏感金额

### 第 19 条 外部沟通限制

- 未经 CFO 批准，不得向外部机构（税务机关除外）披露评估细节
- 与外部税务师事务所的沟通须签 NDA
- IPO / 并购重组的尽调披露按法务建议执行

## 第十章 体系演进

### 第 20 条 年度回顾

每年汇算清缴后 30 天内，加计办主任组织年度政策变动回顾：
- 本年度政策变动汇总
- 公司体系适配情况
- 经验教训
- 下年度政策预期与预案

成果纳入年度评估报告，提交委员会。

### 第 21 条 行业协作

加计办主任应保持与行业协会、税务师事务所、其他企业的政策交流：
- 行业税务师协会年会
- 同行加计扣除经验交流
- 关注典型案例（成功的、被处罚的）
