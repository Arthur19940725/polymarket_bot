| 平台 | 飞书多维表格 + 应用引擎 |

## 第一章 准备工作

### 1.1 账号与权限

- [ ] 加计办主任申请飞书"应用引擎"专业版（500 人规模年费约 8–15 万）
- [ ] CFO 批准飞书订阅升级
- [ ] IT 部门开通飞书 OpenAPI 权限：员工字段读取、消息发送、自动化触发
- [ ] 加计办主任申请"租户管理员"或"应用管理员"权限

### 1.2 工作区结构

在飞书云空间创建顶层目录：
```
研发加计扣除管理/
├── 01-治理（M1 文件归档）
├── 02-判定（M2 文件 + 立项书归档）
├── 03-证据（M3 文件 + 数据归档）
├── 04-内控（M4 文件）
├── 05-培训资料
├── 06-委员会档案
└── 07-数据归档（月度 CSV）
```

### 1.3 字段命名规范

- 全部使用中文（便于业务方理解）
- 主键字段统一带"ID"或"编码"后缀
- 时间戳字段统一"时间"后缀
- 工号字段统一"工号"后缀（不直接用员工 ID 避免歧义）

## 第二章 创建多维表格

### 2.1 新建多维表格

1. 进入飞书云空间「研发加计扣除管理 / 03-证据」目录
2. 点击「新建」→「多维表格」
3. 命名：**「研发加计扣除运行底表」**
4. 设置封面图（可选公司 LOGO）

### 2.2 创建 6 张数据表（按《数据模型说明书》）

在新建的多维表格里，依次添加以下 6 张数据表：

| 序号 | 表名 | 来源参考 |
|------|------|----------|
| 1 | 项目主数据 | 说明书 §二 |
| 2 | 人员-项目映射 | 说明书 §三 |
| 3 | 工时填报 | 说明书 §四 |
| 4 | 产出物登记 | 说明书 §五 |
| 5 | Jira 镜像 | 说明书 §六 |
| 6 | Git 镜像 | 说明书 §七 |

**操作**：每张表点击「+」→「新建数据表」→ 按说明书字段定义逐一添加列

## 第三章 字段配置详解（关键表）

### 3.1 表③「工时填报」核心字段配置

| 字段名 | 字段类型 | 配置要点 |
|--------|----------|----------|
| 员工工号 | 单行文本 | 默认填充：当前登录用户工号（自动化规则）|
| 日期 | 日期 | 默认值：今日；范围限制：今日 - 7 天 ~ 今日 |
| 项目编码 | 单选 | 选项动态从表①引用（仅 A 类 + B 类通过的项目）|
| 工时（小时）| 数字 | 范围 0.5–8.0；步长 0.5 |
| 工作内容描述 | 多行文本 | 字数限制：≥20 字（用自动化规则校验）|
| 关联 Jira Ticket ID | 单行文本 | 格式校验：正则 `^[A-Z]+-\d+$` |
| 关联 Git Commit Hash | 单行文本 | 格式校验：正则 `^[a-f0-9]{7,40}$` |
| 关联会议纪要 ID | 单行文本 | 飞书会议记录 ID |
| 主管复核状态 | 单选 | 待复核（默认）/ 通过 / 退回 |
| 月度合规状态 | 公式 | 见下文 §4.2 |

### 3.2 表①「项目主数据」字段配置

| 字段名 | 字段类型 | 配置要点 |
|--------|----------|----------|
| 项目编码 | 单行文本 | 唯一约束 |
| 项目类别 | 单选 | A / B / B-否决 / C / D（颜色标识）|
| 立项申请书 PDF | 附件 | 限制 PDF 格式，大小 ≤20MB |
| 技术不确定性说明 | 多行文本 | 字数 ≥500（A 类）/ ≥800（B 类）|
| 状态 | 单选 | 立项 / 进行 / 暂停 / 已结项 / 已终止（带颜色）|

## 第四章 自动化规则配置

### 4.1 工时填报合规校验（前 3 个月仅预警）

**自动化场景 1**：员工填报工时时校验 ⭐ 三选一证据

操作步骤：
1. 在表③点击「自动化」→「新建自动化」
2. 触发条件：当记录新增 / 修改
3. 条件判断：`Jira Ticket ID 为空 AND Git Commit Hash 为空 AND 会议纪要 ID 为空`
4. 执行动作（**M3–M5 过渡期**）：
   - 给该记录的"月度合规状态"字段标记为「🟡 黄灯-无客观证据」
   - 发送飞书消息给员工："您的工时记录缺少客观证据，过渡期内仅预警；M6 起将被自动剔除"

**M6 切换后**：
- 同一规则的执行动作改为：「拒绝保存」+ 提示信息

### 4.2 月度合规状态公式

在表③的「月度合规状态」字段（公式类型），输入：

```
IF(
  AND(ISBLANK([关联 Jira Ticket ID]), ISBLANK([关联 Git Commit Hash]), ISBLANK([关联会议纪要 ID])),
  "🟡 无客观证据",
  IF([工时] > 8, "🔴 单日超限",
    IF([主管复核状态] = "退回", "🔴 复核退回",
      IF([主管复核状态] = "通过", "🟢 通过", "⚪ 待复核")
    )
  )
)
```

### 4.3 人员当月项目数 > 3 预警

**自动化场景 2**：表②「人员-项目映射」预警兼岗

1. 触发条件：每月 5 日 09:00 定时
2. 条件判断：`该员工当月项目数 > 3`
3. 执行动作：
   - 该员工当月所有映射记录的"兼岗预警"字段标 🟡
   - 发送飞书消息给对应 PM："员工 XXX 本月参与项目数 ___，请出具《兼岗合理性说明》"

### 4.4 单人单月工时低且占比 100% 预警

**自动化场景 3**

1. 触发条件：每月归集 T+2 日定时
2. 条件判断：`员工当月总工时 < 100h AND 项目分摊占比 = 100%`
3. 执行动作：
   - 该记录标记 🔴
   - 抄送加计办主任 + PM 强制下调比例

## 第五章 视图配置

### 5.1 加计办仪表盘视图

在多维表格添加「仪表盘」（飞书新增功能）：

**核心卡片**：

| 卡片 | 数据源 | 计算 |
|------|--------|------|
| 当月研发人员数 | 表② | 去重员工工号 |
| 当月工时总量 | 表③ | SUM(工时) |
| 客观证据覆盖率 | 表③ | COUNT(三选一非空) / COUNT(全部) |
| 当月红黄灯数 | 表③ | 按合规状态分组计数 |
| 进行中 A 类项目数 | 表① | COUNT(类别=A AND 状态=进行) |

### 5.2 各项目 PM 视图

为每个 PM 创建过滤视图：仅显示其管理项目的工时记录、人员映射。

**操作**：表③ → 「视图」→「+」→ 设置过滤条件 `项目编码 IN (该 PM 负责项目列表)` → 命名"我的项目工时"

### 5.3 员工视图

设置员工权限：只能看到 `员工工号 = 当前用户工号` 的自身工时记录。

## 第六章 接口配置（M4 启动）

### 6.1 HR 月度 CSV 导入

操作步骤（每月 1 日）：
1. HR 接口人在 HR 系统导出员工主数据 CSV（含工号 / 姓名 / 部门 / 在职状态 / 月度工资基数）
2. 脱敏处理：去除身份证号、手机号等无关字段
3. 加计办在飞书多维表格 → 表② / HR 子表 →「导入」→ 选择 CSV →按工号匹配更新

### 6.2 Jira API 接入

操作步骤：
1. IT 在 Jira Admin 创建 Service Account，生成 API Token（只读权限）
2. 在飞书自动化助手新建工作流：
   - 触发：每日 02:00 定时
   - 步骤 1：HTTP GET `https://【Jira 域名】/rest/api/3/search?jql=updated>=-1d&maxResults=1000`
   - 步骤 2：解析 JSON，提取 ticket 字段
   - 步骤 3：写入表⑤「Jira 镜像」（按 Ticket ID 去重）
3. 测试：手动触发一次，验证 50+ 条 ticket 入库

### 6.3 Git Webhook 接入

操作步骤：
1. 飞书自动化助手生成 Webhook URL
2. IT 在 GitLab/GitHub 项目设置中添加 Webhook：
   - URL：飞书提供的端点
   - 事件：Push、Merge Request
   - Secret Token：与飞书侧约定，用于鉴权
3. 飞书自动化解析 Webhook payload，提取 commit 字段
4. 写入表⑥「Git 镜像」
5. 测试：在测试仓库做 1 次 commit，验证 1 分钟内入库

## 第七章 权限设置

### 7.1 角色与权限

按《数据模型说明书》§九 配置：

1. 创建飞书群组：
   - 「研发加计扣除管理-加计办」
   - 「研发加计扣除管理-PM」
   - 「研发加计扣除管理-委员会」
   - 「研发加计扣除管理-财务」
   - 「研发加计扣除管理-内审」

2. 在多维表格右上角「分享」→「成员管理」：
   - 加计办群：可管理（admin）
   - PM 群：可编辑（按项目）
   - 委员会 / 财务 / 内审群：可查看
   - 全员：通过具体表的视图权限单独配置

### 7.2 字段级权限

某些敏感字段（如工资基数）仅加计办 + 财务可见：
- 在该字段右键「设置字段权限」→ 限制部分角色不可见

## 第九章 运维与故障处置

### 9.1 常见故障

| 故障现象 | 排查 |
|----------|------|
| Jira 同步失败 | 检查 API Token 是否过期；网络连通；Jira 版本兼容 |
| Git Webhook 没数据 | 检查 GitLab/GitHub 侧 Webhook 状态；飞书端 Secret Token |
| 工时填报报错 | 检查飞书规则配置；员工权限；项目编码是否在表② |
| 自动化规则未触发 | 检查规则启用状态；触发条件；执行动作是否正确 |

### 9.2 异常告警

- 配置飞书机器人：连续 3 天 Jira/Git 同步失败 → 飞书消息给加计办主任 + IT 接口人
- 月度归集前 1 天自动给 PM 群发提醒"未完成复核"
