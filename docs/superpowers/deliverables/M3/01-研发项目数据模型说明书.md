## 一、数据模型总览

### 1.1 四张主表 + 两张镜像表

```
┌─────────────────────────────────────────────────────┐
│  ① 项目主数据  ← 主表，所有数据的根                  │
│       │                                              │
│       ├── ② 人员-项目映射（月度维护）                │
│       ├── ③ 工时填报（日填）                          │
│       └── ④ 产出物登记                                │
└─────────────────────────────────────────────────────┘
                    ▲
                    │
┌───────────────────┴─────────────────────────────────┐
│  ⑤ Jira 镜像（每日同步）│ ⑥ Git 镜像（Webhook 推送）│
│  ←→ 与表③通过 Ticket ID │ ←→ 与表③通过 Commit Hash  │
└─────────────────────────────────────────────────────┘
```

### 1.2 主键与外键

| 字段 | 类型 | 用途 |
|------|------|------|
| 项目编码 `RD-YYYY-NNN` | 主键 | 表①主键；表②③④⑤⑥外键 |
| 员工工号 | 主键 | HR 主数据；表②③外键 |
| 月份 `YYYY-MM` | 联合主键 | 表②按月维护 |
| 日期 `YYYY-MM-DD` | 联合主键 | 表③按日填报 |
| Jira Ticket ID | 外键 | 表③↔⑤ |
| Git Commit Hash | 外键 | 表③↔⑥ |
| 成本中心代码 | 外键 | 表①↔财务 ERP |

| 员工姓名 | 文本 | ✅ | 自动同步 |

| 月份 | 文本 | ✅ | `YYYY-MM` 格式 |

### 3.2 唯一约束

- (员工工号, 项目编码, 月份) 三字段联合唯一

### 3.3 工时占比校验

- 同一员工当月所有项目占比之和应 ≤ 100%
- 超过 100% 系统报错（飞书自动化规则触发）

### 4.2 校验规则

| 规则 | 触发 | 处置 |
|------|------|------|
| 工时 > 8h/日 | 填报时 | 报错，要求加班审批单附件 |
| 工作描述 < 20 字 | 填报时 | 报错，要求补充 |
| Jira/Git/会议三项全空 | 填报时 | 🟡 黄灯，过渡期 M3–M5 仅预警；M6 起拒绝保存 |
| 项目编码不在表②映射 | 填报时 | 报错，要求 PM 先维护映射 |
| 日期 > 今日 | 填报时 | 报错，禁止未来填报 |
| 日期 < 今日 - 7 天 | 填报时 | 报错，禁止超期补填 |

### 4.3 合规状态机

```
新填报 → 待复核
    ↓ 主管签字
  通过 → 月初校验
            ↓
       ┌────┴────┐
       ▼         ▼
     🟢通过    🟡/🔴异常
                ↓
           PM 修正 / 补证据
                ↓
           月底自动剔除（强制期）
```

## 六、表 ⑤ · Jira 镜像（系统自动同步）

### 6.1 字段定义

| 字段名 | 类型 | 来源 |
|--------|------|------|
| Ticket ID | 文本（主键）| Jira API |
| Jira Project Key | 文本 | Jira API |
| Summary | 文本 | Jira API |
| Status | 单选 | Jira API（Open / In Progress / Done 等）|
| Type | 单选 | Story / Task / Bug / Subtask |
| Assignee 工号 | 文本 | Jira API（需做账号映射）|
| Reporter 工号 | 文本 | Jira API |
| 创建时间 | 日期时间 | Jira API |
| 最后更新时间 | 日期时间 | Jira API |
| 同步时间戳 | 日期时间 | 飞书自动化 |

### 6.2 同步规则

- 每日 02:00 通过 Jira REST API 拉取全部 ticket
- 增量同步：仅拉取 `updated > 上次同步时间 - 1 天` 的 ticket
- 异常告警：连续 3 天同步失败时发飞书消息给加计办

- 失败重试：3 次，指数退避
- 异常告警：单日推送 < 历史均值 50% 时发飞书消息给加计办

## 八、跨表关联示例

### 8.1 计算某项目某月可加计工时

```
SELECT
  t1.项目编码,
  SUM(t3.工时) AS 月度工时
FROM 表① t1
JOIN 表③ t3 ON t1.项目编码 = t3.项目编码
WHERE t1.项目类别 IN ('A', 'B-通过')
  AND t3.合规状态 IN ('🟢通过')
  AND t3.日期 BETWEEN '2026-08-01' AND '2026-08-31'
GROUP BY t1.项目编码
```

### 8.2 计算某员工某月分摊工资

```
月度可加计工资 = 当月工资基数 × ∑(各 A/B 项目实际工时) / 当月总工时
```

### 8.3 客观证据覆盖率

```
覆盖率 = COUNT(工时记录 WHERE Jira IS NOT NULL OR Git IS NOT NULL OR 会议 IS NOT NULL)
         / COUNT(全部工时记录)
```

## 十、数据备份与审计

### 10.1 备份策略

- 飞书多维表格自动云端备份（飞书侧）
- 加计办每月 1 日导出全部六张表 CSV，归档至飞书云空间`/02-数据归档/YYYY-MM/`
- 每年汇算清缴后做整年快照，纳入留存备查档案包

### 10.2 审计日志

- 飞书多维表格内建编辑历史（≥30 天）
- 重要操作（工时剔除、PM 复核退回、项目类别变更）自动记录到`审计日志`表
- 内审季度抽查可调阅
