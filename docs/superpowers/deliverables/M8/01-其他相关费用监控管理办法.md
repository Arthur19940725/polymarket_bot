## 第一章 总则

**第 1 条 目的**
财税〔2015〕119 号 + 国税总局公告 2017 年第 40 号规定，"其他相关费用"加总不得超过研发费用总额的 5%，**超出部分不予加计扣除**。本办法用于明确"其他相关费用"6 类的口径、建立月度监控机制、设置预警与处置流程，避免突破 5% 红线导致超限剔除。

**第 2 条 紧迫性**
Phase 1 评估报告显示，公司当前基线占比 **4.2%**，距离 5% 红线仅剩 0.8 个百分点。如不主动管控，按业务自然增长趋势，Q2–Q3 极有可能超限，直接损失节税金额。

**第 3 条 适用范围**
适用于公司本部全部 A 类、B 类（委员会通过）研发项目的"其他相关费用"归集。

## 第二章 "其他相关费用"6 类定义

### 第 4 条 政策口径

根据财税〔2015〕119 号 + 国税总局公告 2017 年第 40 号，"其他相关费用"指**与研发活动直接相关的下列费用**：

| # | 费用类别 | 软件企业典型表现 | 公司当前占比 |
|---|----------|------------------|--------------|
| 1 | 技术图书资料费、资料翻译费 | 论文订阅、技术书籍、专业期刊、外文资料翻译 | 0.3% |
| 2 | 专家咨询费 | 外部技术专家咨询、行业顾问、技术尽调 | 0.5% |
| 3 | 高新科技研发保险费 | 研发活动专项保险（少见）| 0.0% |
| 4 | 研发成果的检索、分析、评议、论证、鉴定、评审、评估、验收费用 | 第三方检测、技术鉴定、专利检索 | 0.4% |
| 5 | 知识产权的申请费、注册费、代理费 | 专利申请费、软著登记费、专利代理所费 | 0.3% |
| 6 | 职工福利费、补充养老保险费、补充医疗保险费 | 研发人员部分（按工时占比分摊）| 0.2% |
| 7 | 差旅费、会议费 | 研发出差、技术会议、学术会议 | 0.9% |
| 8 | **其他与研发活动直接相关的费用**（关键软件企业项）| **云服务器、SaaS 订阅、开发工具授权、外部 API 调用** | **1.6%** |
| | **合计** | | **4.2%** |

> **第 8 项是软件企业的主要超限风险来源**，必须重点管控。

### 第 5 条 不计入"其他相关费用"的支出

下列支出**不属于**第 6 类（其他相关费用），而是直接归入对应类别：
- 研发人员工资 → 人员人工费用
- 直接投入材料 → 直接投入费用
- 研发设备折旧 → 折旧费用
- 软件许可 / 软件开发工具的资本化购置 → 无形资产摊销
- 研发活动中产生的检测费（与第 6.4 项区分）→ 视情况归入直接投入或本类

判断口径：是否构成项目研发的直接成本？是 → 归对应类别；不是直接构成成本但与研发活动直接相关 → 才进入"其他相关费用"。

### 第 6 条 与营业费用的边界

下列支出**不计入研发费用**：
- ❌ 行政人员的差旅、会议
- ❌ 销售部门的差旅、会议
- ❌ 非研发用的云服务器（如客户运行环境、官网）
- ❌ 非研发用的 SaaS（如 CRM、ERP、HR 系统）
- ❌ 公司年会、团建活动
- ❌ 与研发无关的资料

**关键判定**："**直接相关**"是政策原文，"间接相关"或"全公司共用"不得归入。

## 第三章 软件企业第 8 类专项管控（关键章节）

### 第 7 条 云服务费用归集

公司主要使用 AWS / 阿里云 / 腾讯云等公有云服务。按以下规则归集：

#### 7.1 拆分原则

**严格按项目 / 业务标签拆分**，仅"研发用"部分计入研发费用：

| 用途 | 归集 |
|------|------|
| 研发环境（dev / test / staging）| 研发费用 - 其他相关 |
| 生产环境（prod，客户使用）| 营业费用 - 服务器费 |
| 临时研发实验（短期 < 3 个月）| 研发费用 - 其他相关 |
| 长期生产支持 | 营业费用 |
| AI 模型训练（研发项目相关）| 研发费用 - 其他相关 |
| AI 模型推理（线上服务）| 营业费用 |

#### 7.2 拆分操作

由 IT 部门 + 加计办每月协作执行：
- ☐ 在云控制台按 `tag` / `namespace` / `account` 维度拉取分账数据
- ☐ 财务部按拆分明细分两笔入账
- ☐ 拆分依据 + 计算表归档，作为留存备查证据

#### 7.3 拆分依据档案

每月归集时形成《云服务费用拆分说明表》（飞书表），字段：
| 字段 | 示例 |
|------|------|
| 云服务商 | AWS / 阿里云 / 腾讯云 |
| 账单月份 | 2027-02 |
| 总账单金额 | 120 万元 |
| 研发用金额（标签筛选）| 75 万元 |
| 非研发用金额 | 45 万元 |
| 拆分依据 | 按 namespace `rd-dev`、`rd-test`、`research-` 前缀筛选 |
| IT 接口人确认 | ☑ |
| 加计办确认 | ☑ |

### 第 8 条 SaaS 订阅费用归集

软件企业常用的 SaaS 工具按以下分类：

| 工具类型 | 默认归集 | 例外 |
|----------|----------|------|
| Git 代码托管（GitHub、GitLab）| 研发 - 其他相关 | 仅当 100% 研发用 |
| 项目管理（Jira、禅道、Asana）| 研发 - 其他相关 | 全公司用：按研发人员占比分摊 |
| 协作（飞书、钉钉、Slack）| **不计入**（属公司通用办公）| - |
| 设计工具（Figma、Sketch）| 研发 - 其他相关 | 仅研发设计用 |
| 监控（Datadog、Sentry）| 研发 - 其他相关 | 若同时监控生产，按比例拆 |
| AI API（OpenAI、Anthropic）| **看用途**：研发实验 → 研发；生产服务 → 营业费用 | - |
| 文档（Notion、Confluence）| 研发 - 其他相关 | 若全公司用：按研发人员占比 |

### 第 9 条 外部 API 调用费

- 用于研发实验、POC 验证：✅ 研发 - 其他相关
- 用于生产服务（计入产品成本）：❌ 营业费用

## 第四章 月度监控机制

### 第 10 条 监控频率

- **月度监控**（每月归集闭环后）：财务专员出具《月度其他相关费用监控表》
- **季度评审**（仪表盘）：加计办主任向委员会汇报趋势
- **重大事件**（占比突破预警线）：72 小时内出具专题报告

### 第 11 条 预警阈值

| 阈值 | 占比 | 处置 |
|------|------|------|
| 🟢 安全区间 | < 3.5% | 常规监控 |
| 🟡 预警区间 | 3.5% – 4.5% | 加计办与财务专项审视，识别可优化项 |
| 🔴 警戒区间 | 4.5% – 5.0% | 委员会专题决策，启动费用拆分优化 |
| 🚫 红线（法定）| ≥ 5.0% | **超出部分自动剔除**，本年度不可挽回 |

### 第 12 条 监控数据来源

| 来源 | 字段 | 频率 |
|------|------|------|
| 财务 ERP 研发科目 | 6 类费用明细 | 月度 |
| 飞书表「项目费用归集」| 按项目拆分 | 月度 |
| 云服务分账数据 | 按业务标签拆分 | 月度 |
| SaaS 订阅明细 | 按账号 / 部门 | 季度 |

## 第五章 处置流程

### 第 13 条 黄灯（3.5–4.5%）处置流程

```
T+0：月度归集报告显示占比落入黄灯区间
       ↓
T+3：加计办主任 + 财务专员联合审视，识别可优化项
       - 是否有"非研发用"的支出被误计入
       - 是否有云 / SaaS 费用可进一步拆分
       - 是否有部分费用可推迟到次月或次年发生
       ↓
T+5：出具《优化方案备忘》提交财务总监
       ↓
T+7：执行优化（如有），下月监控重点关注
```

### 第 14 条 红灯（4.5–5.0%）处置流程

```
T+0：占比落入红灯区间
       ↓
T+1：加计办主任 24 小时内向 CFO + 委员会报告
       ↓
T+3：委员会专题会议
       - 评估剩余年度的预期累计占比
       - 决定是否暂停部分非紧急的其他相关费用支出
       - 决定是否启动云 / SaaS 大规模拆分专项
       ↓
T+7：启动处置措施
       - 严控新增其他相关费用
       - 加快推进拆分项目
       - 推迟非紧急支出到次年
       ↓
持续监控：每周 1 次小复盘，每两周向委员会汇报
```

### 第 15 条 超限（≥5%）处置

如年内累计已超 5%：
- 超出部分**法定不可加计**，损失已经发生
- 加计办出具《超限事件根因分析报告》提交委员会
- 制定预防机制纳入下年度管理重点
- 必要时纳入相关责任人考核

## 第六章 优化策略

### 第 16 条 三种优化策略

#### 策略 ① 拆分归集（最有效）

把全公司共用的成本按研发用 vs 非研发用拆分：
- 云服务：按 namespace / 标签拆
- SaaS：按账号 / 部门拆
- 协作工具：通常不计入研发（公司通用）
- 研发设备折旧：仅研发专用部分

预期效果：占比下降 0.5–1.5 个百分点。

#### 策略 ② 重新归类

某些支出可重新归类到更合适的费用项：
- "研发咨询费"是否属于"专家咨询费"还是"无形资产摊销"
- "云服务"是否构成"无形资产"（如年度订阅可摊销）
- "差旅费"是否属于直接投入（如客户现场测试）

预期效果：占比下降 0.2–0.5 个百分点。

#### 策略 ③ 时间错峰

把可推迟的支出错峰到次年：
- 非紧急的专家咨询
- 非紧急的会议、培训
- 非必要的资料采购

预期效果：占比下降 0.2–0.3 个百分点（仅当年）。

### 第 17 条 不推荐的"优化"

下列做法**不推荐**，因为可能引发税务风险：
- ❌ 把"研发用"的差旅伪装成营业费用（少计入研发，反而损失更大节税）
- ❌ 把"非研发用"的费用伪装成研发用（虚增研发费用 → 偷税风险）
- ❌ 把当年发生的费用推到次年（违反权责发生制）

## 第七章 项目 PM 的责任

### 第 18 条 PM 在项目预算阶段的责任

PM 在项目立项时（《立项申请书》第 5.1 节预算）应**预估其他相关费用占比**：
- 若预估占比 >4%，加计办提示 PM 优化预算结构
- 若预估占比 >5%，加计办否决预算

### 第 19 条 PM 在项目执行阶段的责任

- 月度费用归集时，PM 复核本项目的其他相关费用是否合理
- 大额支出（>10 万）须由 PM 提前向加计办报备

## 第八章 与其他制度的衔接

### 第 20 条 与判定层

第 8 类"其他与研发活动直接相关的费用"的判定边界，**严格按《研发活动判定矩阵与操作指引》第 5 条**——不属于研发活动的支出不得归入。

### 第 21 条 与证据层

每笔其他相关费用的发生，须有：
- 关联到具体项目编码
- 客观证据（合同 / 发票 / 使用记录）
- 用途说明（必要时）

### 第 22 条 与内控层

- 月度归集 5 项必对中包含其他相关费用占比检验
- 季度仪表盘指标 3（其他相关费用占比）专项跟踪
- 年度自查覆盖此项

## 第九章 附则

**第 23 条** 本办法与公司财务管理制度冲突时，**对研发费用归集口径以本办法为准**。

**第 24 条** 本办法的解释权归委员会，日常解释由加计办主任行使。

**第 25 条** 本办法自 2027-02-01 起生效。
