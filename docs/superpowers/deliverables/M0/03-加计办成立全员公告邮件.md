## 三、对各位同事的具体影响

### 🧑‍💻 研发部全体同事（**重点关注**）

1. **从 M3（2026 年 8 月起）开始**，工时填报正式上线：
   - 填报工具：飞书多维表格「研发加计扣除管理」工作区
   - 填报频率：**每日下班前**填报当日工时
   - **每条工时项必须关联以下至少一项客观证据**：① Jira Ticket ID ② Git Commit Hash ③ 会议纪要 ID
   - 主管须**每周五前**完成本组工时复核签字

2. **过渡期机制：**
   - M3 启动后的前 3 个月（2026-08 至 2026-10）为**过渡期**，无客观证据的工时项**仅黄灯预警、不剔除**
   - **第 4 个月（2026-11）起强制规则生效**——无证据工时项将被月度系统自动剔除

3. 项目经理（PM）须按月维护项目立项书、人员-项目映射、月度工时复核
