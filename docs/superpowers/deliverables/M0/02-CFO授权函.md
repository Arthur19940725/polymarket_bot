## 关于成立"研发费用加计扣除办公室"并启动研发费用加计扣除风险管理体系搭建项目的授权决定

### 致

公司各部门、各业务单元、研发中心全体员工：

### 一、决定事项

经公司管理层会议研究决定，即日起：

1. **批准启动**《研发费用加计扣除风险管理体系搭建项目》（项目编号 RDD-RMS-2026-001），项目周期 24 个月（2026-06-01 至 2028-05-31），分三阶段实施。
2. **批准成立**"研发费用加计扣除管理委员会"，由本人担任组长，研发 VP / HR 总监 / 内审负责人 / 法务总监担任委员。
3. **批准设立**"研发费用加计扣除办公室"（以下简称"加计办"），常设虚拟编制，作为体系建设与日常运行的执行主体。
4. **任命**【___】同志担任加计办主任，向本人直接汇报；任命【___】、【___】、【___】同志分别担任加计办财务专员、研发 PMO 接口人、HR 接口人，各保留原岗位、按规定工作量占比参与加计办工作。

### 二、加计办职权

加计办在体系建设与运行过程中，拥有以下职权，各部门应予以全力配合：

1. **数据调取权**：有权要求 HR / 研发 / 财务 / 法务 / IT 等部门提供研发活动相关的人员、项目、合同、工时、费用、技术产出等全部数据。
2. **流程审核权**：研发项目立项、中检、结项三道闸门评审中具有"必经"地位；委外 / 合作研发合同未经加计办会签不得签署。
3. **工时合规裁定权**：对研发人员工时填报的真实性、客观证据完整性具有合规裁定权，月度归集中可执行无证据工时项剔除。
4. **培训组织权**：有权组织全员强制性培训与考试，并将通过率纳入部门考核。
5. **风险升级权**：发现重大风险事项可直接升级至委员会专题会议处置。

### 三、各部门配合事项

1. **研发部**：
   - 项目经理（PM）应按月维护"人员-项目映射"、按月复核工时填报数据
   - 所有研发项目须按三道闸门规范开展立项、中检、结项
   - 研发人员须按规定日填工时、关联客观证据（Jira ticket / Git commit / 会议纪要至少一项）

2. **人力资源部**：
   - 每月 1 日前向加计办提供员工主数据（工号 / 姓名 / 部门 / 在职状态 / 月度工资基数）
   - HR 接口人参与加计办日常工作

3. **财务部**：
   - 按加计办月度归集报告结果完成成本中心归集与"研发支出"辅助账登记
   - 财务总监参与月度归集报告终审

4. **法务部**：
   - 委外 / 合作研发合同审核需含加计扣除相关条款专项审查
   - 法务总监参加委员会例会

5. **内审部**：
   - 每季度对加计扣除工作执行抽查
   - 每年汇算清缴前 60 天主导专项自查
   - 内审负责人参加委员会例会

6. **IT 部门**：
   - 配合开通 HR / Jira / GitLab 等系统的只读 API 接入
   - 协助加计办在飞书多维表格平台搭建四张主表与自动化规则

### 四、考核与问责

1. 加计办相关 KPI（合规 / 效率 / 风险共 12 项）由本人按季度审视，纳入相关责任人年度绩效考核。
2. 各部门对加计办工作配合不到位、数据不真实、工时填报舞弊等违规行为，按公司《员工奖惩管理办法》从严处理；情节严重者，移交公司纪检部门处理。

### 五、生效日期

本授权函自签发之日起生效，至 2028-05-31 项目结束日自动失效；体系正式运行后将以《研发费用加计扣除管理办法》替代本授权函中的相关规定。
