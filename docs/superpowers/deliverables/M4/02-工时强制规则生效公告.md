## 一、本次切换的两件核心事

### 1. 客观证据"⭐ 三选一"由"软"变"硬"

**过渡期（截至 2026-10-31）**：
- 工时填报时如未关联 Jira / Git / 会议任一证据，系统**仅黄灯预警**，仍可保存
- 月底**不会自动剔除**这些工时

**强制期（自 2026-11-01）**：
- 工时填报时如三项证据**全部为空**：系统**直接拒绝保存** + 弹窗提示
- 月底未及时补救的黄灯历史记录：**自动从加计扣除基数中剔除**，不进入工资分摊计算

### 2. 单项违规直接剔除规则

下列情形系统将**自动剔除**当条工时，无需人工审批：

| 违规情形 | 处置 |
|----------|------|
| 单日工时 > 8h 且无加班审批单 | 剔除超出部分 |
| 项目编码不在你当月的"人员-项目映射"内 | 整条剔除 |
| 关联的 Jira ticket 类型为 `Bug` / `Support` | 整条剔除（非研发活动）|
| 工作内容描述 < 20 字 | 整条剔除 |
| 项目状态为"暂停" / "已结项" / "已终止" | 整条剔除 |

## 四、关于"过渡期 vs 强制期"的常见疑问

### Q1：过渡期的黄灯工时还能救吗？

A：**10-31 前可以救**——补上证据 + 修正描述。**11-01 之后，过渡期遗留的黄灯将在 T+2 自动剔除**。

### Q2：剔除会扣员工工资吗？

A：**不会。** 工资由 HR 按合同发放，与加计扣除基数无关。剔除只影响**该笔工时是否进入公司加计扣除的计算基数**。换句话说：影响的是公司省多少税，不影响员工拿多少钱。

### Q3：突发情况（如生病、紧急出差）漏填怎么办？

A：可以补填**最近 7 天内**的工时。超过 7 天系统不允许补填——这是公司合规要求，不是技术问题。

### Q4：周末加班 / 远程加班，工时如何填？

A：照填，但要走加班审批流程并附审批单。研发活动不分工作日 / 周末。

### Q5：我是新员工，11 月 1 日才入职，怎么办？

A：入职第一周加计办会有专项培训。培训前不强制填报；培训后按强制规则执行。

### Q6：跨年度的长期项目，是不是要从立项重新走？

A：不需要。立项时间不变；只要 2026-11-01 之后的工时按新规则填报即可。
