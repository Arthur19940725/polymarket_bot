## 第一章 总则

**第 1 条 目的**
将研发加计扣除体系运行情况转化为**可视化、可比较、可追溯**的季度仪表盘，作为委员会季度评审会议的核心议程材料。

**第 2 条 核心设计原则**
1. **指标精简**：7 项核心指标，覆盖量级 / 结构 / 风险 / 合规 / 争议五个维度
2. **红黄绿灯**：每项指标对应明确阈值，一目了然
3. **基线测算**：M3–M5 三个月作为基线测算期，先用默认阈值跑、半年后委员会复审调整
4. **趋势对比**：环比 + 同比 + 与全年预算对比

**第 3 条 仪表盘形式**
- 飞书多维表格内嵌「仪表盘」视图
- PPT 版本用于委员会季度评审会议
- Word / PDF 版本用于汇算清缴前归档

## 第二章 七项核心指标定义

### 指标 1：研发费用 / 营业收入 vs 预算偏差（量级指标）

| 字段 | 内容 |
|------|------|
| 计算公式 | (本季度研发费用 / 营业收入) ÷ (年度预算研发费用 / 预算营业收入) − 1 |
| 数据来源 | 财务 ERP（研发费用 + 营业收入），年度预算（财务部）|
| 单位 | % |
| 频率 | 季度 |
| **绿灯** 🟢 | 偏差 ±5% 以内 |
| **黄灯** 🟡 | ±5% – ±15% |
| **红灯** 🔴 | >±15% |
| 黄红灯处置 | 加计办出具《偏差分析报告》 → 财务总监审议 → 必要时调整下季度归集策略 |

### 指标 2：工资薪金占比（结构指标）

| 字段 | 内容 |
|------|------|
| 计算公式 | 工资薪金归集金额 ÷ 研发费用归集总额 |
| 数据来源 | 月度归集报告汇总 |
| 单位 | % |
| 频率 | 季度 |
| **绿灯** 🟢 | 60% – 75%（软件企业典型区间）|
| **黄灯** 🟡 | 50% – 60% 或 75% – 85% |
| **红灯** 🔴 | <50% 或 >85% |
| 黄红灯处置 | <50%：审视工时分摊机制是否漏报；>85%：审视其他费用是否归集完整 |

### 指标 3："其他相关费用"占比（结构指标，**法定红线**）

| 字段 | 内容 |
|------|------|
| 计算公式 | "其他相关费用"金额 ÷ 研发费用归集总额 |
| 数据来源 | 月度归集报告 |
| 单位 | % |
| 频率 | **月度**（仪表盘汇总到季度）|
| **绿灯** 🟢 | <4% |
| **黄灯** 🟡 | 4% – 5% |
| **红灯** 🔴 | ≥5%（**法定上限**，超限部分自动剔除）|
| 黄红灯处置 | 黄灯：财务专员检查云服务 / SaaS / 差旅 / 会议费明细；红灯：立即剔除超出部分 |

> **注**：政策依据为财税〔2015〕119 号 + 国税总局公告 2017 年第 40 号文，"其他相关费用"不得超过研发费用总额的 5%。

### 指标 4：工时无客观证据占比（风险指标）

| 字段 | 内容 |
|------|------|
| 计算公式 | 月度无 Jira/Git/会议关联的工时项 ÷ 月度全部工时项 |
| 数据来源 | 飞书多维表格表③ |
| 单位 | % |
| 频率 | 月度（仪表盘汇总到季度）|
| **绿灯** 🟢 | <5% |
| **黄灯** 🟡 | 5% – 15% |
| **红灯** 🔴 | >15% |
| 过渡期处置 | M3–M5 过渡期：仅观察基线，不计入红黄绿灯 |
| 强制期处置 | M6 起，>15% 触发研发部专项整改 |

### 指标 5：兼岗高风险人员占比（风险指标）

| 字段 | 内容 |
|------|------|
| 计算公式 | 当月项目数 > 3 的研发人员数 ÷ 研发人员总数 |
| 数据来源 | 飞书多维表格表② |
| 单位 | % |
| 频率 | 月度（仪表盘汇总到季度）|
| **绿灯** 🟢 | <10% |
| **黄灯** 🟡 | 10% – 20% |
| **红灯** 🔴 | >20% |
| 黄红灯处置 | 黄灯：要求 PM 出具兼岗合理性说明；红灯：研发 VP 介入审视团队配置 |

### 指标 6：立项 / 结项报告齐备率（合规指标）

| 字段 | 内容 |
|------|------|
| 计算公式 | 有完整立项书 + 结项报告（如已结项）的项目数 ÷ 全部 A/B 类项目数 |
| 数据来源 | 飞书多维表格表① |
| 单位 | % |
| 频率 | 季度 |
| **绿灯** 🟢 | 100% |
| **黄灯** 🟡 | 90% – 100% |
| **红灯** 🔴 | <90% |
| 黄红灯处置 | 黄灯：加计办督办缺失项目；红灯：项目可能无法计入加计扣除，紧急补救 |

### 指标 7：B 类（准研发）项目数 / 总项目数（争议指标）

| 字段 | 内容 |
|------|------|
| 计算公式 | 委员会通过的 B 类项目数 ÷ 全部 A + B 项目数 |
| 数据来源 | 飞书多维表格表① |
| 单位 | % |
| 频率 | 季度 |
| **绿灯** 🟢 | <20% |
| **黄灯** 🟡 | 20% – 40% |
| **红灯** 🔴 | >40% |
| 黄红灯处置 | 黄灯：审视判定矩阵是否需要细化；红灯：边界判定标准存在系统性偏差，需要委员会专题讨论 |

## 第三章 仪表盘呈现样式

### 3.1 概览页（一屏俯瞰）

```
┌──────────────────────────────────────────────────────────────┐
│  YYYY 年 第 Q 季度 研发加计扣除风险仪表盘                       │
├──────────────────────────────────────────────────────────────┤
│  量级指标                                                       │
│   ▪ 研发费用 / 营收 vs 预算偏差            🟢 +2.3%             │
│                                                                 │
│  结构指标                                                       │
│   ▪ 工资薪金占比                            🟢 68.5%             │
│   ▪ 其他相关费用占比                        🟡 4.6%（红线 5%）   │
│                                                                 │
│  风险指标                                                       │
│   ▪ 工时无客观证据占比                      🟢 3.2%              │
│   ▪ 兼岗高风险人员占比                      🟡 14.5%             │
│                                                                 │
│  合规指标                                                       │
│   ▪ 立项 / 结项报告齐备率                   🟢 100%              │
│                                                                 │
│  争议指标                                                       │
│   ▪ B 类项目占比                            🟡 28.6%             │
├──────────────────────────────────────────────────────────────┤
│  本季度本季红灯指标数：0                                          │
│  本季度本季黄灯指标数：3                                          │
│  上季度比较：好转 1 项 / 持平 4 项 / 恶化 2 项                    │
└──────────────────────────────────────────────────────────────┘
```

### 3.2 指标详情页（每项指标一页）

每项指标的详情页包含：
1. 当季实际值 + 红黄绿灯
2. 趋势图（最近 12 个月折线 + 阈值线）
3. 同比 + 环比 + 全年预算对比
4. 黄红灯归因分析（如适用）
5. 整改计划与时限

### 3.3 异常事件页

记录本季度发生的：
- 工时大额剔除事件
- 项目类别裁决争议
- 接口故障 / 数据缺口
- 内审 / 外审整改要求

## 第四章 仪表盘搭建（飞书多维表格）

### 4.1 数据底表

新建飞书多维表格「KPI 月度数据」表，字段：

| 字段名 | 类型 | 来源 |
|--------|------|------|
| 月份 | 文本 | YYYY-MM |
| 研发费用 / 营收 vs 预算偏差 | 数字（%）| 财务 + 加计办 |
| 工资薪金占比 | 数字（%）| 月度归集报告 |
| 其他相关费用占比 | 数字（%）| 月度归集报告 |
| 工时无客观证据占比 | 数字（%）| 飞书表③ |
| 兼岗高风险人员占比 | 数字（%）| 飞书表② |
| 立项 / 结项报告齐备率 | 数字（%）| 飞书表① |
| B 类项目占比 | 数字（%）| 飞书表① |
| 备注 | 长文本 | 加计办主任填 |

### 4.2 红黄绿灯公式

每项指标增加一个"灯色"字段（公式类型），按本文档第二章定义自动判断。

### 4.3 仪表盘视图

- 在表内点击「仪表盘」→「新建仪表盘」
- 添加 7 个指标卡片（数字 + 灯色）
- 添加 7 个趋势折线图（含阈值线）
- 添加 1 个汇总表（季度红黄灯统计）

## 第五章 季度评审会议议程

季度仪表盘是委员会季度评审会议的核心材料。会议议程标准模板：

```
研发加计扣除管理委员会 YYYY 年 第 Q 季度评审会议

时间：YYYY-MM-DD 14:00–16:00
地点：【会议室】 / 飞书视频会议
出席：CFO（组长）+ 研发 VP + HR 总监 + 内审负责人 + 法务总监
列席：加计办主任（秘书）+ 财务总监

议程：
1. 加计办主任：本季度仪表盘汇报（20 分钟）
2. 财务总监：财务视角补充（10 分钟）
3. 研发 VP：研发执行视角补充（10 分钟）
4. 红黄灯指标专项讨论（40 分钟）
   - 黄灯指标整改计划
   - 红灯指标限期整改决议
5. 边界争议案例审议（B 类项目）（20 分钟）
6. 政策跟踪与影响评估（10 分钟）
7. 下季度工作重点（10 分钟）

决议：会议纪要由加计办主任 5 个工作日内出具，CFO 签发后归档。
```

## 第六章 阈值校准机制

### 第 14 条 基线测算期

- 时间：M3–M5（2026-08-01 至 2026-10-31）
- 期间仪表盘**仅记录数据、不评红黄绿灯**
- 加计办主任记录 7 项指标的月度实际值，作为校准依据

### 第 15 条 半年阈值复审

- 时间：M6（2026-11 末）
- 主导：加计办主任向委员会汇报
- 决议：基于基线数据 + 行业经验，**对默认阈值进行第一次校准**
- 输出：仪表盘阈值 v2.0

### 第 16 条 后续年度复审

- 每年 12 月委员会年度评审会议同步评审阈值
- 重大政策变动 / 体系架构变动时启动专项复审

## 第七章 KPI 与考核挂钩

仪表盘 7 项指标与体系运营 KPI 形成映射（详见《KPI 定义与计算手册》）：

| 仪表盘指标 | 对应 KPI | 责任 KPI 主考人 |
|------------|----------|------------------|
| 工时无客观证据占比 | A2 工时客观证据覆盖率 | 加计办主任 |
| 立项 / 结项报告齐备率 | A1 立项书齐备率 + A3 留存备查齐备率 | 加计办主任 |
| 其他相关费用占比 | C2 | 财务总监 |
| 兼岗高风险人员占比 | C3 | 各 PM |
| B 类项目占比 | 整体争议指标 | 委员会 |

红灯指标连续 2 季度未改善的，相关责任人考核扣分。
