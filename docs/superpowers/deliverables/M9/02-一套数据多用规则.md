## 第一章 总则

**第 1 条 目的**
公司同时支持三套政策口径下的研发费用申报（加计扣除 / 高新认定 / 软件企业）。本规则确立"**一套基础数据、三套申报视图**"的设计原则，避免重复维护、降低数据冲突、提升申报效率。

**第 2 条 核心设计原则**

| 原则 | 含义 |
|------|------|
| **数据中性化** | 飞书底表只记原始事实，不预设任何口径偏好 |
| **规则配置化** | 三套口径的差异处理写在"导出规则"层，不污染底表 |
| **可追溯** | 任一申报数据可追溯回底表原始数据 + 应用了哪条规则 |
| **横向可比** | 三套口径下的同一项目数据可立即对齐比对 |

## 第三章 底层 · 数据中性化

### 第 4 条 字段中性化原则

底层飞书表的所有字段保持**真实记录**，不做口径偏向：

| 例 | 错误（口径绑定）| 正确（中性化）|
|----|-----------------|---------------|
| 工资 | "可加计工资" | "工资基数"（不预设是否可加计）|
| 项目类别 | "加计扣除 A 类" | "研发项目类别 A/B"（不绑定加计扣除）|
| 工时 | "已剔除工时" | "工时（含合规状态）"（合规判断在导出层做）|
| 费用 | "5% 限额内费用" | "费用类别 + 金额"（限额计算在导出层做）|

### 第 5 条 底层新增字段（用于差异化标识）

为支持三套口径，底层表新增以下字段（不替代原字段）：

| 表 | 新增字段 | 类型 | 用途 |
|----|----------|------|------|
| 表① 项目主数据 | 是否高新研发项目 | 单选（是 / 否 / 部分） | 高新申报筛选 |
| 表① 项目主数据 | 是否软件研发项目 | 单选（是 / 否 / 部分） | 软件企业申报筛选 |
| 表② 人员-项目映射 | 是否高新科技人员 | ✅ / ✗ | 高新认定科技人员 |
| 表② 人员-项目映射 | 是否软件研发人员 | ✅ / ✗ | 软件企业研发人员 |
| 表③ 工时填报 | 是否境内研发 | ✅ / ✗ | 高新境内 60% 要求 |
| 费用表 | 是否含管理费 | ✅ / ✗ | 加计扣除排除管理费 |
| 费用表 | 是否含股权激励 | ✅ / ✗ | 加计扣除排除股权激励 |

> **重要**：这些字段是"标识字段"，不直接参与计算；计算逻辑写在规则层。

## 第四章 规则层 · 三套口径处理规则

### 第 6 条 加计扣除规则

```
研发费用 = 人员人工 + 直接投入 + 折旧 + 无形资产摊销 + 设计试验费 + 其他相关费用

筛选条件：
- 项目类别 = A 或 委员会通过的 B
- 工时合规状态 = 通过
- 项目状态 = 进行 / 已结项 / 暂停（不含已终止）
- 费用：是否含管理费 = 否
- 费用：是否含股权激励 = 否
- 费用：是否境外个人委托 = 否

特殊处理：
- 委外费用：境内 × 80% / 境外 × 80% + 三分之二限制
- 其他相关费用：限 5%，超过自动剔除
```

### 第 7 条 高新认定规则

```
研发费用 = 加计扣除研发费用 + 股权激励 + 管理费 + 部分其他费用（限额内）

筛选条件：
- 项目"是否高新研发项目" = 是 或 部分
- 工时合规状态 = 通过（同加计扣除）
- 境内研发占比 ≥ 60%

特殊处理：
- 其他相关费用：限 20%（比加计扣除宽松）
- 委外费用：境内 × 80% / 境外 × 80%，海外占比 ≤ 50%
- 含股权激励
- 含与研发直接相关的管理费
```

### 第 8 条 软件企业认定规则

```
研发费用 = 加计扣除研发费用 + 软件研发相关费用

筛选条件：
- 项目"是否软件研发项目" = 是 或 部分
- 软件研发人员占比 ≥ 40%
- 软件研发费用 / 总研发费用 ≥ 80%

特殊处理：
- 自研软件产品销售收入占软件总收入 ≥ 40%（销售侧验证）
- 含部分管理费 / 股权激励
```

## 第五章 三套申报视图设计

### 第 9 条 飞书多维表格视图

在飞书多维表格的"项目费用归集汇总表"中，配置 3 个并列视图：

```
视图 1：加计扣除申报
   筛选：项目类别 ∈ {A, B-通过}
        AND 工时合规状态 = 通过
        AND 费用-是否含管理费 = 否
        AND 费用-是否含股权激励 = 否
   计算字段：
        - 委外费用 × 0.8
        - 其他相关费用计入上限 = MIN(其他相关, 总额 × 5%)
        - 加计扣除可加计金额 = SUM(上述所有项)
   附加列：当前月份 / 累计 / 占比预测

视图 2：高新认定申报
   筛选：项目-是否高新 ∈ {是, 部分}
        AND 工时合规状态 = 通过
   计算字段：
        - 委外费用 × 0.8（含境外 ≤ 50% 限制）
        - 其他费用计入上限 = MIN(其他相关, 总额 × 20%)
        - 高新研发费用 = SUM(上述 + 股权激励 + 管理费)
        - 高新技术产品收入占比

视图 3：软件企业申报
   筛选：项目-是否软件 ∈ {是, 部分}
        AND 软件研发人员
   计算字段：
        - 软件研发费用
        - 软件产品销售收入占比
        - 自研软件占比
```

### 第 10 条 视图导出

每个视图可独立导出 CSV / Excel，作为对应申报材料的底稿。导出文件命名：
- `加计扣除YYYY年度.csv`
- `高新认定YYYY年度.csv`
- `软件企业YYYY年度.csv`

## 第六章 数据维护流程

### 第 11 条 月度维护流程

```
每月归集（按《月度归集 SOP》）：

T+1：HR/Jira/Git 数据归集
T+2：合规校验（红黄灯）
T+3：PM 复核
T+4：工资分摊
T+5：归集报告签发
       ↓
   ⭐ 同步在三套视图中验证数据：
       - 加计扣除可加计金额
       - 高新认定研发费用
       - 软件企业研发费用
       ↓
   出具三套口径月度对照表（飞书自动生成）
```

### 第 12 条 季度横向比对

每季度末，加计办主任出具《三套口径横向比对季度报告》：
- 总额差异
- 关键项目分类一致性
- 差异原因分类
- 重大冲突点

提交委员会季度评审会议。

### 第 13 条 年度审计前比对

每年 3 月（年审前）做完整比对：
- 加计扣除全年总额
- 高新全年总额
- 软件企业全年总额
- 差异说明
- 各申报的可用数据完整性

为外审审计师提供清晰的数据图谱。

## 第七章 数据冲突预警

### 第 14 条 自动冲突检测规则

在飞书多维表格配置自动化规则：

| 规则 | 触发 | 通知 |
|------|------|------|
| 项目类别 = A 但 "是否高新研发" = 否 | 立即触发 | 加计办主任 |
| "是否高新研发人员" = 是 但 当月工时占比 < 30% | 月度触发 | 加计办主任 |
| 加计扣除研发费用 > 高新研发费用 | 月度触发 | 加计办（异常）|
| 三套口径下某项目缺失 | 季度触发 | 加计办 + 战略部 |

### 第 15 条 重大冲突处置

发现"加计扣除 A 类"但"高新非研发"等重大冲突时：
- 5 个工作日内组织评审（加计办 + 法务 + 战略部）
- 决定调整方向：以哪套口径为准？另一套作"差异说明"
- 委员会备案

## 第八章 文档化与归档

### 第 16 条 归档要求

每年年报 / 高新认定 / 加计扣除申报后，归档：
- 当年度三套口径汇总报表（PDF）
- 差异说明（PDF）
- 横向比对季度报告（4 份 PDF）
- 申报视图导出 CSV（3 份）

保管期 10 年（与留存备查档案同步）。

### 第 17 条 数据权限

- 底层数据：加计办 + 财务 + HR + 战略部可访问（按需）
- 申报视图：加计办 + 财务总监可访问
- 横向比对报告：委员会 + 加计办 + 财务总监 + 战略部

## 第九章 IT 系统设计

### 第 18 条 飞书多维表格的实施

技术实现要点：
1. 底层 6 张表（详见《数据模型说明书》）
2. 在表①、表②等扩展字段（详见 §三第 5 条）
3. 创建 3 个汇总表对应三套申报
4. 配置自动化规则做冲突预警
5. 配置季度自动报告生成

### 第 19 条 未来升级（Phase 3）

随着数据量增长（项目 >50 个、员工 >1000 人），飞书多维表格可能力不从心。Phase 3 评估升级路径：
- 建设独立 BI 系统（如 PowerBI / Tableau / 自研）
- 接入数据中台
- 自动化三套申报材料生成
